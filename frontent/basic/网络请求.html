网络请求 ​

基本概念和内容 ​

网络请求是客户端与服务器之间进行数据交互的过程。在网络请求中，常用的一些概念和内容包括：

请求头（Request Headers） ​

请求头是客户端发送给服务器的一组键值对，用于描述请求的信息，其中包括请求方式、请求路径、请求参数、请求体、身份认证等内容。

常用请求头字段 ​

字段名说明示例值
Content-Type指示请求体的数据类型application/json、application/x-www-form-urlencoded、multipart/form-data
Authorization身份认证凭证Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
User-Agent客户端标识Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Accept客户端可接受的响应类型application/json, text/html, */*
Accept-Encoding支持的压缩编码gzip, deflate, br
Accept-Language首选语言zh-CN,zh;q=0.9,en;q=0.8
Cache-Control缓存控制策略no-cache、max-age=3600
Cookie客户端存储的 CookiesessionId=abc123; userId=456
Referer请求来源页面https://example.com/login
Host请求的主机名和端口api.example.com:8080
Origin请求来源（跨域时重要）https://example.com
Content-Length请求体长度（字节）1024
Connection连接管理keep-alive、close
X-Requested-With请求类型标识（通常为 XMLHttpRequest）XMLHttpRequest

请求头示例 ​

http
GET /api/users HTTP/1.1
Host: api.example.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Accept: application/json
Accept-Encoding: gzip, deflate
Accept-Language: zh-CN,zh;q=0.9
Cache-Control: no-cache

Authorization 认证类型 ​

Authorization 请求头支持多种认证方案，常见的类型如下：

类型格式说明安全性
BearerBearer <token>OAuth 2.0 令牌，最常用高
BasicBasic <base64(username:password)>基础认证，用户名密码 Base64 编码低（需 HTTPS）
API KeyApi-Key <key> 或自定义API 密钥认证中（需保密）
DigestDigest <response>摘要认证，比 Basic 安全中
HawkHawk <credentials>基于 HMAC 的认证高
AWS4-HMAC-SHA256AWS4-HMAC-SHA256 <params>AWS 签名认证高
1. Bearer Token（最常用） ​

OAuth 2.0 标准认证方式，用于传递访问令牌（Access Token）：

http
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c

结构：

  • Header：算法和令牌类型（如 {"alg":"HS256","typ":"JWT"}）
  • Payload：用户信息和声明
  • Signature：签名验证
2. Basic Auth（基础认证） ​

将用户名和密码用冒号连接后进行 Base64 编码：

http
# 原始凭证：username:password
# Base64 编码后：dXNlcm5hbWU6cGFzc3dvcmQ=
Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=

特点：

  • 简单但不安全，必须配合 HTTPS 使用
  • 浏览器会弹出登录对话框
3. API Key（API 密钥） ​

直接使用预分配的密钥进行认证：

http
# 方式1：使用 Authorization 头
Authorization: Api-Key sk-1234567890abcdef1234567890abcdef

# 方式2：使用自定义请求头
X-API-Key: sk-1234567890abcdef1234567890abcdef

# 方式3：放在查询参数中（不推荐）
GET /api/users?api_key=sk-1234567890abcdef

适用场景：

  • 服务器间通信
  • 第三方集成
4. Digest Auth（摘要认证） ​

比 Basic 更安全，不传输明文密码：

http
Authorization: Digest username="admin", realm="example.com", nonce="dcd98b7102dd2f0e8b11d0f600bfb0c093", uri="/api/users", qop=auth, nc=00000001, cnonce="0a4f113b", response="6629fae49393a05397450978507c4ef1", opaque="5ccc069c403ebaf9f0171e9517f40e41"

特点：

  • 使用 MD5/SHA 哈希算法
  • 避免明文传输密码
  • 比 Basic 安全，但不如 Bearer Token
5. AWS 签名认证 ​

AWS 服务使用的认证方式：

http
Authorization: AWS4-HMAC-SHA256 Credential=AKIAIOSFODNN7EXAMPLE/20130524/us-east-1/s3/aws4_request, SignedHeaders=host;range;x-amz-date, Signature=fe5f80f77d5fa3beca038a248ff027d0445342fe2855ddc963176630326f1024

组成部分：

  • Credential：访问密钥 ID + 日期 + 区域 + 服务 + 请求类型
  • SignedHeaders：签名的请求头列表
  • Signature：计算后的签名
认证类型选择建议 ​
场景推荐认证方式说明
Web 应用用户认证Bearer Token (JWT)标准 OAuth 2.0 流程
内部系统/服务器通信API Key简单直接
遗留系统Basic Auth (HTTPS)兼容性好
AWS 服务AWS Signature官方推荐
高安全要求场景OAuth 2.0 + PKCE增强安全性

请求体（Request Body） ​

请求体用于在 HTTP 请求中向服务端传递数据，例如在 POST 或 PUT 方法中提交表单数据或上传文件时，就需要使用请求体。

常见请求体格式 ​

格式类型Content-Type适用场景特点
JSONapplication/json结构化数据传输可读性好，支持复杂数据结构
表单编码application/x-www-form-urlencoded简单表单提交键值对格式，不支持文件
表单数据multipart/form-data文件上传、复杂表单支持二进制文件和多部分数据
纯文本text/plain简单文本数据无格式限制
XMLapplication/xml传统 API 数据交换结构化标记语言

请求体示例 ​

JSON 格式：

json
{
  "name": "张三",
  "age": 28,
  "email": "zhangsan@example.com",
  "address": {
    "city": "广州",
    "district": "天河区"
  },
  "tags": ["前端", "JavaScript", "Vue"]
}

表单编码格式：

name=张三&age=28&email=zhangsan%40example.com&city=%E5%B9%BF%E5%B7%9E

表单数据格式（multipart/form-data）：

http
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW

------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="username"

张三
------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="avatar"; filename="avatar.jpg"
Content-Type: image/jpeg

[二进制数据]
------WebKitFormBoundary7MA4YWxkTrZu0gW--

响应体（Response Body） ​

响应体是服务器返回给客户端的处理结果，包含了请求所需要的数据或者错误信息。

常见响应体格式 ​

格式类型Content-Type适用场景
JSONapplication/jsonAPI 接口响应
HTMLtext/html网页内容
XMLapplication/xml传统 API 响应
图片image/jpeg、image/png、image/gif图片资源
二进制application/octet-stream文件下载
CSStext/css样式文件
JavaScriptapplication/javascript脚本文件

响应体示例 ​

成功响应（JSON）：

json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "张三",
    "email": "zhangsan@example.com",
    "createdAt": "2024-01-15T10:30:00Z"
  },
  "pagination": {
    "total": 100,
    "page": 1,
    "size": 10
  }
}

错误响应（JSON）：

json
{
  "code": 400,
  "message": "请求参数错误",
  "errors": [
    {
      "field": "email",
      "message": "邮箱格式不正确"
    },
    {
      "field": "password",
      "message": "密码长度不能少于6位"
    }
  ]
}

HTML 响应：

html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>首页</title>
</head>
<body>
  <h1>欢迎访问我的网站</h1>
  <p>这是一个示例页面</p>
</body>
</html>

请求方法（Request Methods） ​

请求方法指示客户端对服务器资源的操作类型。

方法说明是否有请求体幂等性
GET获取资源否是
POST创建新资源是否
PUT更新完整资源是是
PATCH部分更新资源是否
DELETE删除资源否是
HEAD获取响应头（无响应体）否是
OPTIONS获取服务器支持的方法否是
CONNECT建立隧道连接否否
TRACE回显请求否是

幂等性：多次执行相同请求，结果相同。

请求路径（Request URL） ​

请求路径指示客户端请求的资源在服务器上的位置。它由以下部分组成：

https://example.com:8080/api/users?id=123&name=张三#section1
\____/   \__________/ \__/ \______/ \_______________/ \______/
  |         |          |      |           |            |
协议      主机名      端口   路径      查询参数      锚点

查询参数（Query Parameters） ​

查询参数是附加在请求路径后面的键值对，用于传递额外的请求参数。

查询参数示例 ​

http
GET /api/users?page=1&size=10&sort=createdAt&order=desc HTTP/1.1
Host: api.example.com

参数说明：

  • page=1：页码，从第1页开始
  • size=10：每页10条数据
  • sort=createdAt：按创建时间排序
  • order=desc：降序排列

状态码（Status Codes） ​

状态码是服务器在响应中返回的一个数字，用于表示请求的处理结果。

状态码分类 ​

分类范围说明
1xx100-199信息性响应，请求已接收，继续处理
2xx200-299成功响应，请求已成功处理
3xx300-399重定向，需要进一步操作
4xx400-499客户端错误，请求有问题
5xx500-599服务器错误，服务器处理失败

常用状态码 ​

状态码名称说明
200OK请求成功
201Created资源创建成功
204No Content请求成功但无响应体
301Moved Permanently永久重定向
302Found临时重定向
304Not Modified资源未修改（缓存有效）
400Bad Request请求参数错误
401Unauthorized未经授权，需要身份认证
403Forbidden禁止访问（无权限）
404Not Found请求资源不存在
405Method Not Allowed请求方法不允许
408Request Timeout请求超时
409Conflict资源冲突（如重复创建）
422Unprocessable Entity请求体格式正确但内容无效
429Too Many Requests请求过于频繁（限流）
500Internal Server Error服务器内部错误
502Bad Gateway网关错误
503Service Unavailable服务不可用
504Gateway Timeout网关超时

响应头（Response Headers） ​

响应头是服务器在响应中返回给客户端的一组键值对，用于描述响应的信息。

常用响应头字段 ​

字段名说明示例值
Content-Type响应体数据类型application/json; charset=utf-8
Content-Length响应体长度2048
Content-Encoding响应体压缩编码gzip
Cache-Control缓存策略max-age=3600, public
Expires缓存过期时间Thu, 15 Apr 2025 20:00:00 GMT
Last-Modified资源最后修改时间Wed, 10 Jan 2024 15:30:00 GMT
ETag资源标识（缓存验证）"abc123"
Set-Cookie设置 CookiesessionId=abc123; HttpOnly; Secure
Access-Control-Allow-Origin跨域允许的来源*、https://example.com
Access-Control-Allow-Methods跨域允许的方法GET, POST, PUT, DELETE
Access-Control-Allow-Headers跨域允许的请求头Content-Type, Authorization
Location重定向目标 URLhttps://example.com/new-path
Server服务器标识nginx/1.20.0
X-Powered-By服务器技术栈Express、PHP/7.4

响应头示例 ​

http
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Content-Length: 256
Content-Encoding: gzip
Cache-Control: max-age=3600
ETag: "abc123"
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Server: nginx/1.20.0
