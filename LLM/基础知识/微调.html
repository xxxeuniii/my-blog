大模型微调 ​

什么是微调 ​

微调（Fine-tuning）是指在预训练大语言模型的基础上，使用特定领域或任务的数据集进行进一步训练，使模型适应特定任务的过程。

微调的目的 ​

  1. 领域适配：将通用模型调整为特定领域（如医疗、法律、金融）的专家模型
  2. 任务定制：让模型掌握特定任务（如文本分类、命名实体识别、摘要生成）
  3. 性能提升：在特定任务上获得更好的精度和效果
  4. 个性化：让模型具备特定风格或知识

微调方法分类 ​

全参数微调（Full Fine-tuning） ​

对模型的所有参数进行更新训练。

优点：

  • 效果最好，充分利用预训练知识
  • 适合数据充足的场景

缺点：

  • 计算资源消耗大
  • 训练时间长
  • 需要大量高质量标注数据

适用场景：

  • 有充足计算资源（GPU/TPU）
  • 有大量高质量标注数据
  • 对模型效果要求极高

参数高效微调（Parameter-Efficient Fine-Tuning, PEFT） ​

只更新模型的一小部分参数，保持大部分预训练参数不变。

1. LoRA（Low-Rank Adaptation） ​

在 Transformer 的注意力层中注入低秩矩阵。

核心思想：

  • 冻结预训练模型参数
  • 在注意力权重矩阵旁边添加低秩分解矩阵
  • 只训练低秩矩阵的参数

优势：

  • 参数量减少 100-1000 倍
  • 训练速度快，显存占用低
  • 多个任务可以共享基础模型

代码示例：

python
from peft import LoraConfig, get_peft_model
import transformers

model = transformers.AutoModelForCausalLM.from_pretrained("base-model")

lora_config = LoraConfig(
    r=8,  # 秩
    lora_alpha=32,  # 缩放因子
    target_modules=["q_proj", "v_proj"],  # 目标模块
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)
model.print_trainable_parameters()

2. QLoRA（Quantized LoRA） ​

在 LoRA 基础上加入量化技术，进一步降低显存占用。

特点：

  • 将模型量化为 4-bit 或 8-bit
  • 在量化后的模型上应用 LoRA
  • 支持在单张消费级 GPU 上微调大模型

代码示例：

python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM, BitsAndBytesConfig

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16
)

model = AutoModelForCausalLM.from_pretrained(
    "base-model",
    quantization_config=bnb_config,
    device_map="auto"
)

lora_config = LoraConfig(
    r=64,
    lora_alpha=16,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)

3. Prompt Tuning ​

冻结模型参数，只训练提示词嵌入。

特点：

  • 在输入层添加可训练的虚拟 tokens
  • 适用于多任务场景
  • 训练速度极快

代码示例：

python
from peft import PromptTuningConfig, get_peft_model

prompt_config = PromptTuningConfig(
    task_type="CAUSAL_LM",
    num_virtual_tokens=20,
    prompt_tuning_init="TEXT",
    prompt_tuning_init_text="分类任务：",
    tokenizer_name_or_path="tokenizer"
)

model = get_peft_model(model, prompt_config)

4. Prefix Tuning ​

在 Transformer 的每一层添加可训练的前缀向量。

特点：

  • 比 Prompt Tuning 效果更好
  • 支持更长的上下文

5. Adapter Tuning ​

在 Transformer 层之间插入小型网络模块。

特点：

  • 每个 Transformer 层添加一个适配器
  • 不影响原有模型结构

微调流程 ​

1. 数据准备 ​

数据格式 ​

  • 监督微调（SFT）：输入-输出对
  • 指令微调：指令-输入-输出三元组
  • 对比学习：正例和负例对

数据示例 ​

json
{
  "instruction": "将以下文本分类为正面、负面或中性：",
  "input": "这部电影太棒了，演员表演非常出色！",
  "output": "正面"
}

数据预处理 ​

python
def preprocess_function(examples):
    inputs = [f"指令：{instr}\n输入：{inp}" for instr, inp in zip(examples["instruction"], examples["input"])]
    model_inputs = tokenizer(inputs, max_length=512, truncation=True)
    
    with tokenizer.as_target_tokenizer():
        labels = tokenizer(examples["output"], max_length=128, truncation=True)
    
    model_inputs["labels"] = labels["input_ids"]
    return model_inputs

2. 模型选择 ​

模型类型适用场景推荐模型
通用对话聊天机器人LLaMA-2、Qwen、Mistral
代码生成代码补全CodeLlama、StarCoder
多模态图文理解GPT-4V、Qwen-VL
专业领域医疗/法律领域微调模型

3. 训练配置 ​

超参数设置 ​

python
training_args = TrainingArguments(
    output_dir="./results",
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    num_train_epochs=3,
    logging_steps=10,
    save_strategy="epoch",
    fp16=True,
    report_to="none"
)

关键超参数说明 ​

参数说明推荐值
learning_rate学习率1e-4 ~ 3e-4
batch_size批次大小视显存调整
epochs训练轮数2 ~ 5
weight_decay权重衰减0.01
warmup_ratio预热比例0.05 ~ 0.1

5. 模型评估 ​

自动评估指标 ​

  • BLEU：文本生成质量
  • ROUGE：摘要质量
  • Accuracy：分类任务
  • Perplexity：语言模型困惑度

人工评估 ​

  • 样本抽样检查
  • 对比基线模型
  • 用户测试

微调技巧 ​

数据质量 ​

  • 使用高质量、领域相关的数据
  • 数据清洗和去重
  • 数据增强（回译、同义替换）

学习率调度 ​

  • 预热阶段（warmup）
  • 余弦衰减（cosine decay）
  • 线性衰减

正则化 ​

  • Dropout
  • 权重衰减
  • 梯度裁剪

混合精度训练 ​

  • FP16/FP8 训练
  • 减少显存占用
  • 加速训练

常见问题 ​

过拟合 ​

表现：训练集效果好，验证集效果差

解决方法：

  • 增加训练数据
  • 提前停止（early stopping）
  • 增加正则化
  • 使用数据增强

模式崩溃 ​

表现：模型输出重复或无意义内容

解决方法：

  • 调整学习率
  • 增加训练数据多样性
  • 使用更大的批次

训练不稳定 ​

表现：损失波动大，NaN 值

解决方法：

  • 降低学习率
  • 增加梯度累积
  • 检查数据质量

实战案例：微调 LLaMA-2 做情感分析 ​

数据准备 ​

python
import json

data = [
    {"text": "这部电影很棒！", "label": "positive"},
    {"text": "服务很差，不会再来了", "label": "negative"},
    {"text": "产品一般，价格适中", "label": "neutral"}
]

with open("sentiment_data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False)

加载模型和数据 ​

python
from datasets import load_dataset
from transformers import AutoTokenizer, AutoModelForSequenceClassification

tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-hf")
model = AutoModelForSequenceClassification.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    num_labels=3
)

dataset = load_dataset("json", data_files="sentiment_data.json")

配置 LoRA ​

python
from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="SEQ_CLS"
)

model = get_peft_model(model, lora_config)

训练 ​

python
training_args = TrainingArguments(
    output_dir="./sentiment-model",
    per_device_train_batch_size=2,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    num_train_epochs=3
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset["train"],
    eval_dataset=dataset["train"]
)

trainer.train()

总结 ​

微调是将通用大模型适配到特定任务的关键技术。选择合适的微调方法需要考虑：

  • 可用计算资源
  • 数据量和质量
  • 任务复杂度
  • 部署约束

推荐从 QLoRA 开始尝试，它在效果和效率之间取得了很好的平衡。
