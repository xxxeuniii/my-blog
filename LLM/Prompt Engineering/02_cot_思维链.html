COT (Chain of Thought) 思维链 ​

让模型一步步思考，适合复杂推理问题。

示例一：数学应用题 ​

问题：小明有10个苹果，送给小红3个，又买了5个，现在有几个？

让我们一步步思考：
1. 小明开始有10个苹果
2. 送给小红3个：10 - 3 = 7个
3. 又买了5个：7 + 5 = 12个
答案是：12个

示例二：逻辑推理 ​

问题：所有人都会死，苏格拉底是人，所以？

让我们一步步推理：
1. 前提1：所有人都会死（大前提）
2. 前提2：苏格拉底是人（小前提）
3. 根据三段论：苏格拉底也会死
答案是：苏格拉底也会死

示例三：代码调试 ​

问题：以下Python代码为什么会报错？

代码：
numbers = [1, 2, 3, 4, 5]
print(numbers[10])

让我们分析：
1. 列表 numbers 有5个元素，索引为 0-4
2. 访问 numbers[10] 是在访问第11个元素
3. 列表中不存在这个索引
4. Python会抛出 IndexError
答案：因为索引越界，列表最大索引是4，但访问了索引10

示例四：商业决策分析 ​

问题：一家奶茶店月收入下降20%，分析原因

让我们一步步分析：
1. 收入下降的可能因素：
   - 客流量减少
   - 客单价降低
   - 竞争加剧
   - 产品问题
2. 逐一排查：
   - 上月是否有新开竞争对手？→ 是，街对面新开一家
   - 本月是否有负面评价？→ 是，有几条关于卫生的差评
   - 价格是否有变化？→ 无
3. 结论：主要原因是竞争加剧和个别差评影响口碑
建议：加强卫生管理，可考虑推出优惠活动吸引顾客

适用场景 ​

场景效果
数学计算减少计算错误
逻辑推理理清思路
代码调试定位根因
商业分析全面考虑因素
决策判断权衡利弊

核心要点 ​

  • 要求模型"一步步思考"
  • 在 prompt 中加入"让我们分析"或"think step by step"
  • 鼓励模型展示推理过程
  • 复杂问题可要求列出"可能原因 → 逐一验证 → 结论"

进阶技巧 ​

1. 指定思考步骤 ​

请分析这个问题，每一步都要给出理由：
"是否应该购买电动汽车？"

请按以下步骤分析：
1. 列出购买电动汽车的优点
2. 列出购买电动汽车的缺点
3. 对比分析
4. 根据不同人群给出建议

2. 自我验证 ​

问题：2673 × 482 = ?

请计算并验证：
1. 第一次计算
2. 用除法验证结果
3. 如果不一致，重新计算

3. 反向思考 ​

问题：为什么这个创业项目会失败？

让我们从多个角度分析：
1. 市场角度：需求是否真实存在？
2. 产品角度：是否真正解决了问题？
3. 团队角度：能力是否匹配？
4. 资金角度：现金流是否能支撑？
5. 竞争角度：护城河在哪里？
