# 评估与迭代

建立测试集、LLM打分、版本对比的完整流程。

---

## 为什么需要评估

```
没有评估，就没有优化方向：
- 不知道哪种Prompt更好
- 不知道效果下降还是上升
- 版本混乱难以回滚
```

---

## 第一步：建立测试集

```
测试集构成：
1. 代表性问题（覆盖主要场景）
2. 基准答案（或质量标准）
3. 评分维度（明确评估标准）

示例测试集（10个问题）：
Q1. 情感分类：用户评论是正面/负面/中性？
Q2. 代码生成：写一个Python排序函数
Q3. 信息提取：从邮件提取会议时间地点
Q4. 翻译：中文→英文技术文档翻译
...
```

---

## 第二步：LLM自动打分

```
用LLM做评估器的Prompt：

你是一个严谨的AI评估员。请从以下维度评估回答质量：

维度：
1. 正确性（0-5分）：回答是否准确
2. 完整性（0-5分）：是否覆盖所有要点
3. 清晰度（0-5分）：表达是否清晰
4. 格式（0-5分）：是否符合要求格式

请返回JSON：
{
  "total_score": 总分,
  "dimensions": {
    "correctness": 分数,
    "completeness": 分数,
    "clarity": 分数,
    "format": 分数
  },
  "feedback": "改进建议"
}
```

---

## 第三步：版本对比

```
建立Prompt版本管理：

v1: 基础版，不加例子
v2: 加入2个Few-shot例子
v3: 加入COT，让模型分步思考
v4: 格式要求更明确

对比维度：
| 版本 | 平均分数 | Token消耗 | 延迟 |
|------|----------|----------|------|
| v1 | 2.3分 | 120 | 2s |
| v2 | 3.8分 | 250 | 2.5s |
| v3 | 4.5分 | 320 | 3s |
| v4 | 4.8分 | 350 | 3.2s |

选择：v4性价比最高，选v4
```

---

## 第四步：迭代优化循环

```
1. 观察问题 → 2. 提出假设 → 3. 修改Prompt → 4. 测试验证 → 5. 评估决策 → 回1

示例：
观察：复杂数学题正确率只有50%
假设：加入COT（一步步思考）可以改善
修改：在Prompt中加入"请逐步思考"
测试：用测试集重新打分
决策：COT让正确率提升到85%，正式采用
```

## 快速评估模板

```
简易版本：直接让LLM对比两个回答哪个更好

Prompt：
回答A：[旧版本输出]
回答B：[新版本输出]

哪个回答更好？请说明原因并给出分数（A/B）。
```
