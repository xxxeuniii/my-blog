# Few-shot Prompting（少样本提示）

给模型几个例子，让它学习模式。

---

## 示例一：情感分类

```
请判断以下评论的情绪是正面、负面还是中性：

例子1: "这家餐厅的披萨太好吃了！" → 正面
例子2: "等了2小时才上菜，差评。" → 负面
例子3: "环境不错，但价格偏贵。" → 中性

请判断："耳机音质一般，没有宣传的那么好"
```

---

## 示例二：文本纠错

```
请修正以下句子中的语法错误：

例子1: "我昨天去了商店，买了一个苹果和三个面包。" → "我昨天去了商店，买了一个苹果和三块面包。"
例子2: "他不应该这么着急，应该慢慢来。" → "他不应该这么着急，应该慢慢来。"（无需修改）

请修正："小明和小红是一对好朋友，他们天天一起上学和玩耍"
```

---

## 示例三：风格转换

```
将以下句子转换为正式商务邮件风格：

例子1: "嗨，这个方案你看下，有问题跟我说" → "尊敬的[姓名]，您好，现将方案发送给您，请查阅。如有任何问题，请随时与我联系。"
例子2: "赶紧把报告交了，领导催呢！" → "请尽快提交报告，领导已在催促。感谢您的配合。"

请转换："兄弟，那个合同你签了没？赶紧弄完发我"
```

---

## 适用场景

- 任务明确但答案多样
- 需要模型理解特定格式或风格
- 罕见任务需要示例引导

## 核心要点

| 要点 | 说明 |
|------|------|
| 数量 | 2-5 个代表性例子 |
| 多样性 | 例子要覆盖常见类型 |
| 一致性 | 格式和风格保持统一 |
| 顺序 | 简单 → 复杂 排列 |

## 进阶技巧

### 1. 零样本 + 少样本结合

```
请判断以下评论的情感：

"这家店的服务态度太差了，等了半小时没人理"

情感：负面

现在请判断：
"终于收到了，质量还不错，就是发货有点慢"
```

### 2. 正面、负面、中性 全覆盖

```
请判断新闻标题的语气是正面、负面还是中性：

正面："我国科技创新取得重大突破"
负面："某地发生严重交通事故，致多人伤亡"
中性："今日股市收盘点位"

请判断："央行宣布降准0.5个百分点"
```

### 3. 复杂输出格式示例

```
请将以下产品描述转换为指定格式：

输入："iPhone 15 Pro Max，256GB，深空灰色，支持5G，售价9999元"

输出格式：
{
  "产品名": "xxx",
  "配置": ["xxx", "xxx"],
  "颜色": "xxx",
  "网络": "xxx",
  "价格": xxx
}

输入："华为Mate60，512GB，黑色，支持卫星通话，售价6999元"
```
