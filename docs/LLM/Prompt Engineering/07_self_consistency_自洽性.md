# Self-Consistency (自洽性)

多路径推理，取最一致的答案。

---

## 示例一：数学计算验证

```
问题：一家商店原价200元的商品，打8折后再降价15%，最终价格是多少？

请用两种方法计算：

方法1：先打折后降价
1. 原价200元
2. 打8折：200 × 0.8 = 160元
3. 再降15%：160 × (1 - 0.15) = 160 × 0.85 = 136元

方法2：计算整体折扣
1. 打8折 = 80%
2. 再降15% = 85%
3. 整体折扣 = 80% × 85% = 68%
4. 最终价格 = 200 × 0.68 = 136元

两种方法结果一致：136元
结论：答案可信，最终价格为136元
```

## 示例二：事实核查

```
问题：珠穆朗玛峰的高度是？

请从多个角度验证：

来源1：教科书 → 8848米
来源2：维基百科 → 8848.86米（2020年最新测量）
来源3：旅游网站 → 8848米

分析：
- 不同来源数据略有差异
- 原因是测量技术和标准不同
- 岩石高度 vs 雪顶高度

结论：珠穆朗玛峰高度约为8848米（岩石高度），最新测量为8848.86米（含雪顶）
```

## 示例三：代码逻辑验证

```
问题：这段代码的输出是什么？

代码：
result = 0
for i in range(1, 5):
    result += i
print(result)

验证1 - 累加计算：
1 + 2 + 3 + 4 = 10

验证2 - 数学公式：
等差数列求和：(1 + 4) × 4 / 2 = 10

验证3 - 代码执行：
print(0 + 1 + 2 + 3 + 4) = 10

结论：输出结果为 10，三种方式一致
```

## 适用场景

| 场景 | 效果 |
|------|------|
| 重要计算 | 金融、医学、 工程计算 |
| 事实核查 | 避免单一来源错误 |
| 代码调试 | 多角度验证逻辑 |
| 关键决策 | 降低错误风险 |

## 核心要点

- 同一问题用不同方法/路径解决
- 比较结果是否一致
- 一致 → 高可信度
- 不一致 → 需要进一步分析

## 实际应用建议

```
对于重要任务，可以这样要求：
"请用两种不同的方法计算这个结果，
如果两种方法结果一致，说明答案可信；
如果不一致，请分析原因并给出最可能的答案"
```
