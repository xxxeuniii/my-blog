# 提示词压缩与缓存

Token优化、降低延迟、成本控制。

---

## 为什么需要优化Token

```
成本因素：
- Token按使用量计费
- 大上下文窗口会显著增加成本

性能因素：
- Token越多，生成越慢
- 响应延迟增加用户体验差

实际场景：
- 一个复杂Prompt可能用掉5000+ tokens
- 1M tokens约$5
- 每天1000次请求 → $25/天 → $750/月
```

## 技巧一：压缩Prompt

```
原始版（啰嗦）：
你是一个非常专业的而且经验丰富的数据分析师，请帮我仔细地分析一下这些数据，然后给出详细的建议...

压缩版（简洁）：
你是数据分析师，请分析数据并给出建议。
```

---

## 技巧二：移除冗余说明

```
❌ 冗余：
你要帮我做文本分类。类别有：正面、负面、中性。
正面是指...负面是指...中性是指...

✅ 精简：
文本分类，返回正面/负面/中性。
```

## 技巧三：重用组件

```
将常用的角色设定、格式要求做成模板，动态插入。

模板示例：
{
  "role": "你是[角色]",
  "format": "返回JSON：{\"result\": ...}"
}
```

## 技巧四：缓存机制

```
相同问题 → 直接返回缓存结果

缓存策略：
1. 精确匹配：问题完全一样
2. 语义缓存：相似问题返回相似答案
3. TTL：缓存过期时间（如1小时）
4. 优先级：高频问题优先缓存

实现示例：
from cachetools import TTLCache
cache = TTLCache(maxsize=1000, ttl=3600)  # 1小时过期

def get_response(prompt):
    if prompt in cache:
        return cache[prompt]
    response = llm.generate(prompt)
    cache[prompt] = response
    return response
```

## 技巧五：流式输出（降低感知延迟）

```
API调用：
openai.ChatCompletion.create(
  stream=True,
  ...
)

用户体验：
- 普通模式：等3秒 → 完整显示
- 流式模式：0.2秒 → 逐字显示（虽然总时间一样，但感觉快很多）
```

## 综合优化策略

```
1. 设计阶段优化
   - Prompt简洁直接
   - 避免冗余描述

2. 运行阶段优化
   - 实现缓存
   - 优先用小模型处理简单任务

3. 长期优化
   - 定期清理不用的缓存
   - 分析Token使用报表
   - 持续压缩优化
```
