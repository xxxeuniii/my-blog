# Prompt Chaining (提示链)

将复杂任务拆分为多个步骤，逐步完成。

---

## 示例一：文章处理流水线

```
第一步 - 文章总结：
请总结以下文章的核心内容（3个要点）：
"人工智能技术的快速发展正在深刻改变各行各业..."

第二步 - 提取关键词：
基于刚才的总结，提取5个关键词

第三步 - 生成标题：
根据总结和关键词，生成3个吸引人的文章标题

第四步 - 撰写摘要：
将总结扩展成一段100字的文章摘要
```

---

## 示例二：用户评论分析

```
第一步 - 情感分类：
判断以下评论的情感（正面/负面/中性）：
"等了半小时才上菜，菜品还有点凉"

第二步 - 关键信息提取：
从评论中提取：问题类型、等待时间、具体不满点

第三步 - 严重程度评估：
基于提取的信息，评估这个问题的严重程度（1-5分）

第四步 - 生成回复：
针对这条评论，生成一条合适的商家回复
```

---

## 示例三：需求文档生成

```
第一步 - 需求收集：
从用户描述中提取功能需求：
"我希望有一个功能能自动备份手机照片到云端，并且能按时间分类"

第二步 - 需求分析：
将需求拆分为：
- 核心功能（必须）
- 附加功能（应该有）
- 扩展功能（可以有）

第三步 - 编写PRD：
基于以上分析，撰写产品需求文档，包含：
- 产品概述
- 功能列表
- 用户场景
- 非功能需求

第四步 - 技术评估：
评估实现难度和周期
```

## 适用场景

| 场景 | 优势 |
|------|------|
| 复杂长任务 | 拆解后更清晰 |
| 多阶段处理 | 每步专注单一任务 |
| 流水线生产 | 结果可复用、可调整 |
| 质量要求高 | 每步可审核、修正 |

## 核心要点

| 要点 | 说明 |
|------|------|
| 单一职责 | 每步只做一个明确任务 |
| 上下文传递 | 前一步输出作为后一步输入 |
| 可独立优化 | 某步效果不好可单独改进 |
| 结果验证 | 中间步骤可检查、修正 |

## 注意事项

- 每步指令要清晰明确
- 适当在prompt中传递前一步的结果
- 如果某步出错，可以单独重试
- 不要拆得太细，反而增加复杂度

## Prompt Chaining vs ReAct

| 方法 | 区别 |
|------|------|
| Chaining | 按顺序执行，线性流程 |
| ReAct | 推理+行动+观察，循环迭代 |
