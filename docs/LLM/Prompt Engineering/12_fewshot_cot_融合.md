# Few-shot + COT 融合

既给例子，又让模型一步步思考。

---

## 示例一：情感分析

```
请分析以下评论的情感，并展示推理过程。

例子：
"这个电影太精彩了，笑中带泪" → 正面
推理：评论中使用了"精彩"、"笑中带泪"等积极词汇，表达了对电影的赞赏。

"等了2小时电影才开始，差评" → 负面
推理：评论中提到"等2小时"表示不满，"差评"直接表达了负面情感。

请分析：
"前半部分还行，后半部分太拖沓了"
```

## 示例二：数学应用题

```
请解答以下问题，并展示完整推理过程。

例子：
"小明有10元，买了3本练习本，每本2元，还剩多少？"
解：
1. 总钱数：10元
2. 花费：3 × 2 = 6元
3. 剩余：10 - 6 = 4元
答案：4元

请解答：
"小红有25元，买了一支钢笔8元，又买了3支铅笔，每支3元，还剩多少？"
```

## 示例三：代码审查

```
请审查以下代码，识别问题并展示分析过程。

例子：
代码：
def divide(a, b):
    return a / b

审查过程：
1. 检查函数逻辑：直接返回a除以b
2. 潜在问题：如果b为0会抛出ZeroDivisionError
3. 严重程度：高
4. 建议：添加除零检查

请审查：
代码：
def find_max(scores):
    return max(scores)
```

## 适用场景

| 场景 | 效果 |
|------|------|
| 复杂格式要求 | 既给格式例子，又让展示过程 |
| 推理类任务 | 例子演示推理步骤 |
| 专业领域 | 专业判断标准和过程 |
| 教学场景 | 展示思考过程 |

## 核心要点

| 要点 | 说明 |
|------|------|
| 例子包含过程 | 不仅给答案，要展示"为什么" |
| 格式一致 | 例子的格式要和期望输出一致 |
| 引导词 | 用"推理过程"、"分析步骤"等引导 |
| 渐进复杂 | 例子从简单到复杂 |
