# Temperature (温度参数)

控制输出随机性和创造力的参数。

---

## 参数解释

```
Temperature = 0.0 ~ 2.0（通常）

低温度 (0.0 ~ 0.3)：
- 输出更确定、更保守
- 适合精确任务：翻译、代码、问答

高温度 (0.7 ~ 1.5)：
- 输出更随机、更有创意
- 适合创意任务：写诗、故事、营销文案

极高温度 (> 1.5)：
- 输出高度随机，可能不稳定
- 慎用，可能产生无意义内容
```

---

## 实际对比示例

### 任务：写产品 slogan

**Temperature = 0（确定）**
```
输入：为一杯奶茶写slogan
输出：天然奶茶，健康之选
```

**Temperature = 0.9（创意）**
```
输入：为一杯奶茶写slogan
输出1：喝出夏威夷的味道
输出2：这一刻，让味蕾去旅行
输出3：不止是一杯奶茶，是生活的小确幸
```

---

### 任务：代码补全

**Temperature = 0（确定）**
```
输入：写一个快速排序函数
输出：结果稳定，每次都一样
```

**Temperature = 0.9（创意）**
```
输入：写一个快速排序函数
输出：每次可能用不同的写法或注释风格
```

## 场景推荐值

| 任务类型 | Temperature | 说明 |
|----------|-------------|------|
| 代码生成 | 0.0 - 0.3 | 精确、稳定 |
| 数据提取 | 0.0 - 0.2 | 严格按照格式 |
| 问答/翻译 | 0.2 - 0.5 | 稳定但自然 |
| 内容创作 | 0.7 - 1.0 | 有创意 |
| 头脑风暴 | 0.8 - 1.2 | 多样化 |
| 角色扮演 | 0.8 - 1.2 | 更生动 |

---

## 注意事项

| 问题 | 说明 |
|------|------|
| 不是越高越好 | 高温度可能产生无意义内容 |
| 结合Top-p | 通常二选一，不要同时调 |
| 多次生成 | 高温度时可用多次生成选最优 |
| 任务匹配 | 精确任务用低温度，创意任务用高温度 |

## 组合使用

```
代码 → Temperature 0.1 + Few-shot
文案 → Temperature 0.8 + Few-shot
分析 → Temperature 0.3 + COT
```
