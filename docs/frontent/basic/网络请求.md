# 网络请求

## 基本概念和内容

网络请求是客户端与服务器之间进行数据交互的过程。在网络请求中，常用的一些概念和内容包括：

### 请求头（Request Headers）

请求头是客户端发送给服务器的一组键值对，用于描述请求的信息，其中包括请求方式、请求路径、请求参数、请求体、身份认证等内容。

#### 常用请求头字段

| 字段名 | 说明 | 示例值 |
|--------|------|--------|
| `Content-Type` | 指示请求体的数据类型 | `application/json`、`application/x-www-form-urlencoded`、`multipart/form-data` |
| `Authorization` | 身份认证凭证 | `Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` |
| `User-Agent` | 客户端标识 | `Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36` |
| `Accept` | 客户端可接受的响应类型 | `application/json, text/html, */*` |
| `Accept-Encoding` | 支持的压缩编码 | `gzip, deflate, br` |
| `Accept-Language` | 首选语言 | `zh-CN,zh;q=0.9,en;q=0.8` |
| `Cache-Control` | 缓存控制策略 | `no-cache`、`max-age=3600` |
| `Cookie` | 客户端存储的 Cookie | `sessionId=abc123; userId=456` |
| `Referer` | 请求来源页面 | `https://example.com/login` |
| `Host` | 请求的主机名和端口 | `api.example.com:8080` |
| `Origin` | 请求来源（跨域时重要） | `https://example.com` |
| `Content-Length` | 请求体长度（字节） | `1024` |
| `Connection` | 连接管理 | `keep-alive`、`close` |
| `X-Requested-With` | 请求类型标识（通常为 XMLHttpRequest） | `XMLHttpRequest` |

#### 请求头示例

```http
GET /api/users HTTP/1.1
Host: api.example.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Accept: application/json
Accept-Encoding: gzip, deflate
Accept-Language: zh-CN,zh;q=0.9
Cache-Control: no-cache
```

#### Authorization 认证类型

`Authorization` 请求头支持多种认证方案，常见的类型如下：

| 类型 | 格式 | 说明 | 安全性 |
|------|------|------|--------|
| **Bearer** | `Bearer <token>` | OAuth 2.0 令牌，最常用 | 高 |
| **Basic** | `Basic <base64(username:password)>` | 基础认证，用户名密码 Base64 编码 | 低（需 HTTPS） |
| **API Key** | `Api-Key <key>` 或自定义 | API 密钥认证 | 中（需保密） |
| **Digest** | `Digest <response>` | 摘要认证，比 Basic 安全 | 中 |
| **Hawk** | `Hawk <credentials>` | 基于 HMAC 的认证 | 高 |
| **AWS4-HMAC-SHA256** | `AWS4-HMAC-SHA256 <params>` | AWS 签名认证 | 高 |

##### 1. Bearer Token（最常用）

OAuth 2.0 标准认证方式，用于传递访问令牌（Access Token）：

```http
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```

**结构**：
- Header：算法和令牌类型（如 `{"alg":"HS256","typ":"JWT"}`）
- Payload：用户信息和声明
- Signature：签名验证

##### 2. Basic Auth（基础认证）

将用户名和密码用冒号连接后进行 Base64 编码：

```http
# 原始凭证：username:password
# Base64 编码后：dXNlcm5hbWU6cGFzc3dvcmQ=
Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=
```

**特点**：
- 简单但不安全，**必须配合 HTTPS 使用**
- 浏览器会弹出登录对话框

##### 3. API Key（API 密钥）

直接使用预分配的密钥进行认证：

```http
# 方式1：使用 Authorization 头
Authorization: Api-Key sk-1234567890abcdef1234567890abcdef

# 方式2：使用自定义请求头
X-API-Key: sk-1234567890abcdef1234567890abcdef

# 方式3：放在查询参数中（不推荐）
GET /api/users?api_key=sk-1234567890abcdef
```

**适用场景**：
- 服务器间通信
- 第三方集成

##### 4. Digest Auth（摘要认证）

比 Basic 更安全，不传输明文密码：

```http
Authorization: Digest username="admin", realm="example.com", nonce="dcd98b7102dd2f0e8b11d0f600bfb0c093", uri="/api/users", qop=auth, nc=00000001, cnonce="0a4f113b", response="6629fae49393a05397450978507c4ef1", opaque="5ccc069c403ebaf9f0171e9517f40e41"
```

**特点**：
- 使用 MD5/SHA 哈希算法
- 避免明文传输密码
- 比 Basic 安全，但不如 Bearer Token

##### 5. AWS 签名认证

AWS 服务使用的认证方式：

```http
Authorization: AWS4-HMAC-SHA256 Credential=AKIAIOSFODNN7EXAMPLE/20130524/us-east-1/s3/aws4_request, SignedHeaders=host;range;x-amz-date, Signature=fe5f80f77d5fa3beca038a248ff027d0445342fe2855ddc963176630326f1024
```

**组成部分**：
- Credential：访问密钥 ID + 日期 + 区域 + 服务 + 请求类型
- SignedHeaders：签名的请求头列表
- Signature：计算后的签名

##### 认证类型选择建议

| 场景 | 推荐认证方式 | 说明 |
|------|-------------|------|
| Web 应用用户认证 | Bearer Token (JWT) | 标准 OAuth 2.0 流程 |
| 内部系统/服务器通信 | API Key | 简单直接 |
| 遗留系统 | Basic Auth (HTTPS) | 兼容性好 |
| AWS 服务 | AWS Signature | 官方推荐 |
| 高安全要求场景 | OAuth 2.0 + PKCE | 增强安全性 |

### 请求体（Request Body）

请求体用于在 HTTP 请求中向服务端传递数据，例如在 POST 或 PUT 方法中提交表单数据或上传文件时，就需要使用请求体。

#### 常见请求体格式

| 格式类型 | Content-Type | 适用场景 | 特点 |
|----------|--------------|----------|------|
| **JSON** | `application/json` | 结构化数据传输 | 可读性好，支持复杂数据结构 |
| **表单编码** | `application/x-www-form-urlencoded` | 简单表单提交 | 键值对格式，不支持文件 |
| **表单数据** | `multipart/form-data` | 文件上传、复杂表单 | 支持二进制文件和多部分数据 |
| **纯文本** | `text/plain` | 简单文本数据 | 无格式限制 |
| **XML** | `application/xml` | 传统 API 数据交换 | 结构化标记语言 |

#### 请求体示例

**JSON 格式**：
```json
{
  "name": "张三",
  "age": 28,
  "email": "zhangsan@example.com",
  "address": {
    "city": "广州",
    "district": "天河区"
  },
  "tags": ["前端", "JavaScript", "Vue"]
}
```

**表单编码格式**：
```
name=张三&age=28&email=zhangsan%40example.com&city=%E5%B9%BF%E5%B7%9E
```

**表单数据格式（multipart/form-data）**：
```http
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW

------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="username"

张三
------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="avatar"; filename="avatar.jpg"
Content-Type: image/jpeg

[二进制数据]
------WebKitFormBoundary7MA4YWxkTrZu0gW--
```

### 响应体（Response Body）

响应体是服务器返回给客户端的处理结果，包含了请求所需要的数据或者错误信息。

#### 常见响应体格式

| 格式类型 | Content-Type | 适用场景 |
|----------|--------------|----------|
| **JSON** | `application/json` | API 接口响应 |
| **HTML** | `text/html` | 网页内容 |
| **XML** | `application/xml` | 传统 API 响应 |
| **图片** | `image/jpeg`、`image/png`、`image/gif` | 图片资源 |
| **二进制** | `application/octet-stream` | 文件下载 |
| **CSS** | `text/css` | 样式文件 |
| **JavaScript** | `application/javascript` | 脚本文件 |

#### 响应体示例

**成功响应（JSON）**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "张三",
    "email": "zhangsan@example.com",
    "createdAt": "2024-01-15T10:30:00Z"
  },
  "pagination": {
    "total": 100,
    "page": 1,
    "size": 10
  }
}
```

**错误响应（JSON）**：
```json
{
  "code": 400,
  "message": "请求参数错误",
  "errors": [
    {
      "field": "email",
      "message": "邮箱格式不正确"
    },
    {
      "field": "password",
      "message": "密码长度不能少于6位"
    }
  ]
}
```

**HTML 响应**：
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>首页</title>
</head>
<body>
  <h1>欢迎访问我的网站</h1>
  <p>这是一个示例页面</p>
</body>
</html>
```

### 请求方法（Request Methods）

请求方法指示客户端对服务器资源的操作类型。

| 方法 | 说明 | 是否有请求体 | 幂等性 |
|------|------|--------------|--------|
| **GET** | 获取资源 | 否 | 是 |
| **POST** | 创建新资源 | 是 | 否 |
| **PUT** | 更新完整资源 | 是 | 是 |
| **PATCH** | 部分更新资源 | 是 | 否 |
| **DELETE** | 删除资源 | 否 | 是 |
| **HEAD** | 获取响应头（无响应体） | 否 | 是 |
| **OPTIONS** | 获取服务器支持的方法 | 否 | 是 |
| **CONNECT** | 建立隧道连接 | 否 | 否 |
| **TRACE** | 回显请求 | 否 | 是 |

> **幂等性**：多次执行相同请求，结果相同。

### 请求路径（Request URL）

请求路径指示客户端请求的资源在服务器上的位置。它由以下部分组成：

```
https://example.com:8080/api/users?id=123&name=张三#section1
\____/   \__________/ \__/ \______/ \_______________/ \______/
  |         |          |      |           |            |
协议      主机名      端口   路径      查询参数      锚点
```

### 查询参数（Query Parameters）

查询参数是附加在请求路径后面的键值对，用于传递额外的请求参数。

#### 查询参数示例

```http
GET /api/users?page=1&size=10&sort=createdAt&order=desc HTTP/1.1
Host: api.example.com
```

**参数说明**：
- `page=1`：页码，从第1页开始
- `size=10`：每页10条数据
- `sort=createdAt`：按创建时间排序
- `order=desc`：降序排列

### 状态码（Status Codes）

状态码是服务器在响应中返回的一个数字，用于表示请求的处理结果。

#### 状态码分类

| 分类 | 范围 | 说明 |
|------|------|------|
| **1xx** | 100-199 | 信息性响应，请求已接收，继续处理 |
| **2xx** | 200-299 | 成功响应，请求已成功处理 |
| **3xx** | 300-399 | 重定向，需要进一步操作 |
| **4xx** | 400-499 | 客户端错误，请求有问题 |
| **5xx** | 500-599 | 服务器错误，服务器处理失败 |

#### 常用状态码

| 状态码 | 名称 | 说明 |
|--------|------|------|
| 200 | OK | 请求成功 |
| 201 | Created | 资源创建成功 |
| 204 | No Content | 请求成功但无响应体 |
| 301 | Moved Permanently | 永久重定向 |
| 302 | Found | 临时重定向 |
| 304 | Not Modified | 资源未修改（缓存有效） |
| 400 | Bad Request | 请求参数错误 |
| 401 | Unauthorized | 未经授权，需要身份认证 |
| 403 | Forbidden | 禁止访问（无权限） |
| 404 | Not Found | 请求资源不存在 |
| 405 | Method Not Allowed | 请求方法不允许 |
| 408 | Request Timeout | 请求超时 |
| 409 | Conflict | 资源冲突（如重复创建） |
| 422 | Unprocessable Entity | 请求体格式正确但内容无效 |
| 429 | Too Many Requests | 请求过于频繁（限流） |
| 500 | Internal Server Error | 服务器内部错误 |
| 502 | Bad Gateway | 网关错误 |
| 503 | Service Unavailable | 服务不可用 |
| 504 | Gateway Timeout | 网关超时 |

### 响应头（Response Headers）

响应头是服务器在响应中返回给客户端的一组键值对，用于描述响应的信息。

#### 常用响应头字段

| 字段名 | 说明 | 示例值 |
|--------|------|--------|
| `Content-Type` | 响应体数据类型 | `application/json; charset=utf-8` |
| `Content-Length` | 响应体长度 | `2048` |
| `Content-Encoding` | 响应体压缩编码 | `gzip` |
| `Cache-Control` | 缓存策略 | `max-age=3600, public` |
| `Expires` | 缓存过期时间 | `Thu, 15 Apr 2025 20:00:00 GMT` |
| `Last-Modified` | 资源最后修改时间 | `Wed, 10 Jan 2024 15:30:00 GMT` |
| `ETag` | 资源标识（缓存验证） | `"abc123"` |
| `Set-Cookie` | 设置 Cookie | `sessionId=abc123; HttpOnly; Secure` |
| `Access-Control-Allow-Origin` | 跨域允许的来源 | `*`、`https://example.com` |
| `Access-Control-Allow-Methods` | 跨域允许的方法 | `GET, POST, PUT, DELETE` |
| `Access-Control-Allow-Headers` | 跨域允许的请求头 | `Content-Type, Authorization` |
| `Location` | 重定向目标 URL | `https://example.com/new-path` |
| `Server` | 服务器标识 | `nginx/1.20.0` |
| `X-Powered-By` | 服务器技术栈 | `Express`、`PHP/7.4` |

#### 响应头示例

```http
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Content-Length: 256
Content-Encoding: gzip
Cache-Control: max-age=3600
ETag: "abc123"
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Server: nginx/1.20.0
```

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

{
  "name": "王五",
