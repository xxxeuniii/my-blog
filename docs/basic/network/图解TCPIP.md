# 图解 TCP/IP

## 第一章：网络基础知识

### 1.1 OSI 参考模型

计算机网络中，**接口**和**协议**是两个核心概念：

- **接口**：上下层之间进行交互时所遵循的约定
- **协议**：同一层之间进行交互时所遵循的约定

#### OSI 七层模型

| 层级 | 名称 | 主要功能 |
| :--- | :--- | :--- |
| **7** | 应用层 | 为应用程序提供网络服务（HTTP、FTP、DNS） |
| **6** | 表示层 | 数据格式转换、加密解密、压缩解压 |
| **5** | 会话层 | 建立、管理和终止表示层实体之间的通信会话 |
| **4** | 传输层 | 端到端的数据传输（TCP、UDP） |
| **3** | 网络层 | 路由选择和分组转发（IP） |
| **2** | 数据链路层 | 物理地址寻址、帧同步、差错检测 |
| **1** | 物理层 | 物理介质上的信号传输（网线、光纤、无线电） |

#### 数据封装过程

当数据从应用层向下传输时，每一层都会添加自己的头部信息：

```
应用层数据
    ↓ 添加应用层头部
应用层协议数据单元 (APDU)
    ↓ 添加表示层头部
表示层协议数据单元 (PPDU)
    ↓ 添加会话层头部
会话层协议数据单元 (SPDU)
    ↓ 添加传输层头部
传输层协议数据单元 (Segment / Datagram)
    ↓ 添加网络层头部
网络层协议数据单元 (Packet)
    ↓ 添加数据链路层头部和尾部
数据链路层协议数据单元 (Frame)
    ↓ 转换为物理信号
物理层传输
```

### 1.2 计算机网络分类

根据覆盖范围，计算机网络可分为：

#### 局域网 (LAN - Local Area Network)

- **覆盖范围**：较小地理范围（如办公室、学校、家庭）
- **特点**：高速、低延迟、成本低
- **典型技术**：以太网、Wi-Fi

#### 广域网 (WAN - Wide Area Network)

- **覆盖范围**：较大地理范围（城市、国家、全球）
- **特点**：覆盖广、可能经过多个网络设备
- **典型技术**：光纤、卫星通信、VPN

#### 城域网 (MAN - Metropolitan Area Network)

- **覆盖范围**：介于局域网和广域网之间（城市范围）
- **用途**：连接多个局域网

## 第二章：TCP/IP 协议族

### 2.1 TCP/IP 四层模型

TCP/IP 协议族简化了 OSI 模型，将其分为四层：

| TCP/IP 层 | 对应 OSI 层 | 主要协议 |
| :--- | :--- | :--- |
| **应用层** | 应用层 + 表示层 + 会话层 | HTTP、HTTPS、FTP、SMTP、DNS |
| **传输层** | 传输层 | TCP、UDP |
| **网络层** | 网络层 | IP、ICMP、ARP、RARP |
| **网络接口层** | 数据链路层 + 物理层 | 以太网、Wi-Fi、PPP |

### 2.2 IP 协议

#### IP 地址

IP 地址用于标识网络中的主机，分为 IPv4 和 IPv6：

**IPv4 地址**：32 位二进制数，通常表示为点分十进制（如 `192.168.1.1`）

**IPv6 地址**：128 位二进制数，通常表示为冒号分隔的十六进制（如 `2001:0db8:85a3:0000:0000:8a2e:0370:7334`）

#### IP 地址分类（IPv4）

| 类别 | 范围 | 用途 |
| :--- | :--- | :--- |
| A | 1.0.0.0 - 126.255.255.255 | 大型网络 |
| B | 128.0.0.0 - 191.255.255.255 | 中型网络 |
| C | 192.0.0.0 - 223.255.255.255 | 小型网络 |
| D | 224.0.0.0 - 239.255.255.255 | 多播 |
| E | 240.0.0.0 - 255.255.255.255 | 保留 |

#### 私有 IP 地址

用于局域网内部，不会在互联网上路由：

- A 类：`10.0.0.0 - 10.255.255.255`
- B 类：`172.16.0.0 - 172.31.255.255`
- C 类：`192.168.0.0 - 192.168.255.255`

### 2.3 TCP 协议

#### TCP 特点

- **面向连接**：建立连接 → 数据传输 → 断开连接
- **可靠传输**：确认、重传、流量控制、拥塞控制
- **有序交付**：按顺序传递数据
- **面向字节流**：无边界的字节序列

#### TCP 三次握手

建立连接的过程：

```
客户端                    服务器
  │                          │
  │  SYN (seq=x)             │
  │─────────────────────────>│
  │                          │
  │  SYN+ACK (seq=y, ack=x+1)│
  │<─────────────────────────│
  │                          │
  │  ACK (ack=y+1)           │
  │─────────────────────────>│
  │                          │
  │      连接建立成功        │
```

#### TCP 四次挥手

断开连接的过程：

```
客户端                    服务器
  │                          │
  │  FIN (seq=x)             │
  │─────────────────────────>│  客户端不再发送数据
  │                          │
  │  ACK (ack=x+1)           │
  │<─────────────────────────│  服务器确认收到
  │                          │
  │  FIN (seq=y)             │
  │<─────────────────────────│  服务器不再发送数据
  │                          │
  │  ACK (ack=y+1)           │
  │─────────────────────────>│  客户端确认收到
  │                          │
  │      连接断开成功        │
```

#### TCP 滑动窗口

用于流量控制，提高传输效率：

- **发送窗口**：允许发送但尚未确认的数据范围
- **接收窗口**：接收方当前能接收的数据量
- **拥塞窗口**：根据网络拥塞情况动态调整

### 2.4 UDP 协议

#### UDP 特点

- **无连接**：不需要建立连接
- **不可靠传输**：不保证数据到达，不保证顺序
- **面向报文**：保留消息边界
- **开销小**：头部只有 8 字节

#### UDP vs TCP

| 特性 | TCP | UDP |
| :--- | :--- | :--- |
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠（重传、确认） | 不可靠 |
| 顺序 | 保证顺序 | 不保证顺序 |
| 速度 | 较慢 | 较快 |
| 开销 | 较大（20-60 字节头部） | 较小（8 字节头部） |
| 适用场景 | 文件传输、网页浏览 | 实时视频、音频、游戏 |

## 第三章：常用应用层协议

### 3.1 HTTP 协议

#### HTTP 请求方法

| 方法 | 作用 |
| :--- | :--- |
| GET | 请求获取资源 |
| POST | 提交数据到服务器 |
| PUT | 更新资源 |
| DELETE | 删除资源 |
| HEAD | 获取资源头部信息 |
| OPTIONS | 获取服务器支持的方法 |

#### HTTP 状态码

| 状态码 | 类别 | 说明 |
| :--- | :--- | :--- |
| 1xx | 信息性 | 请求已接收，继续处理 |
| 2xx | 成功 | 请求成功处理 |
| 3xx | 重定向 | 需要进一步操作 |
| 4xx | 客户端错误 | 请求有误 |
| 5xx | 服务器错误 | 服务器处理失败 |

### 3.2 HTTPS 协议

在 HTTP 基础上加入 SSL/TLS 加密：

- **加密方式**：非对称加密（握手）+ 对称加密（数据传输）
- **证书验证**：通过 CA 证书验证服务器身份
- **端口**：默认 443

### 3.3 DNS 协议

域名系统（Domain Name System）：

- **功能**：将域名转换为 IP 地址
- **查询方式**：递归查询、迭代查询
- **记录类型**：A、AAAA、CNAME、MX、TXT

## 第四章：网络层辅助协议

### 4.1 ARP 协议

地址解析协议（Address Resolution Protocol）：

- **功能**：将 IP 地址转换为 MAC 地址
- **工作方式**：广播请求，单播响应

### 4.2 ICMP 协议

互联网控制消息协议（Internet Control Message Protocol）：

- **功能**：传递控制消息和错误报告
- **常见应用**：ping 命令

### 4.3 DHCP 协议

动态主机配置协议（Dynamic Host Configuration Protocol）：

- **功能**：自动分配 IP 地址、子网掩码、网关、DNS 服务器
- **过程**：发现 → 提供 → 请求 → 确认

---

## 第五章：网络设备

### 5.1 交换机 (Switch)

- **工作层次**：数据链路层
- **功能**：基于 MAC 地址转发帧
- **特点**：分割冲突域，不分割广播域

### 5.2 路由器 (Router)

- **工作层次**：网络层
- **功能**：基于 IP 地址转发数据包
- **特点**：分割广播域，实现不同网络之间的通信

### 5.3 网关 (Gateway)

- **功能**：连接不同协议的网络
- **作用**：协议转换、数据格式转换

---

## 总结

TCP/IP 协议族是互联网的基石，理解其分层结构和各层协议的作用，对于深入理解网络通信原理至关重要：

1. **分层设计**：每层有明确的职责，通过接口与上下层交互
2. **协议栈**：从应用层到物理层，数据逐层封装后传输
3. **核心协议**：IP 负责寻址，TCP 负责可靠传输，UDP 负责快速传输
4. **辅助协议**：ARP、ICMP、DHCP 等协议协助完成网络通信

掌握这些基础知识，是学习网络编程、网络安全、系统架构的重要基础。
