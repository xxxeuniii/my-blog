# 计算机网络基础

## 一、TCP/IP 协议栈

### OSI 七层模型

| 层级 | 名称 | 主要功能 |
| :--- | :--- | :--- |
| 7 | 应用层 | 为应用程序提供网络服务（HTTP、FTP、DNS） |
| 6 | 表示层 | 数据格式转换、加密解密、压缩解压 |
| 5 | 会话层 | 建立、管理和终止表示层实体之间的通信会话 |
| 4 | 传输层 | 端到端的数据传输（TCP、UDP） |
| 3 | 网络层 | 路由选择和分组转发（IP） |
| 2 | 数据链路层 | 物理地址寻址、帧同步、差错检测 |
| 1 | 物理层 | 物理介质上的信号传输 |

### TCP/IP 四层模型

| TCP/IP 层 | 对应 OSI 层 | 主要协议 |
| :--- | :--- | :--- |
| 应用层 | 应用层 + 表示层 + 会话层 | HTTP、HTTPS、FTP、SMTP、DNS |
| 传输层 | 传输层 | TCP、UDP |
| 网络层 | 网络层 | IP、ICMP、ARP |
| 网络接口层 | 数据链路层 + 物理层 | 以太网、Wi-Fi |

## 二、IP 协议

### IPv4 地址分类

| 类别 | 范围 | 用途 |
| :--- | :--- | :--- |
| A | 1.0.0.0 - 126.255.255.255 | 大型网络 |
| B | 128.0.0.0 - 191.255.255.255 | 中型网络 |
| C | 192.0.0.0 - 223.255.255.255 | 小型网络 |
| D | 224.0.0.0 - 239.255.255.255 | 多播 |

### 私有 IP 地址

- A 类：`10.0.0.0 - 10.255.255.255`
- B 类：`172.16.0.0 - 172.31.255.255`
- C 类：`192.168.0.0 - 192.168.255.255`

## 三、TCP 协议

### 特点

- 面向连接：建立连接 → 数据传输 → 断开连接
- 可靠传输：确认、重传、流量控制、拥塞控制
- 有序交付：按顺序传递数据

### 三次握手

1. 客户端 → 服务器：SYN
2. 服务器 → 客户端：SYN + ACK
3. 客户端 → 服务器：ACK

### 四次挥手

1. 客户端 → 服务器：FIN
2. 服务器 → 客户端：ACK
3. 服务器 → 客户端：FIN
4. 客户端 → 服务器：ACK

## 四、UDP 协议

### 特点

- 无连接：不需要建立连接
- 不可靠传输：不保证数据到达，不保证顺序
- 面向报文：保留消息边界
- 开销小：头部只有 8 字节

### UDP vs TCP

| 特性 | TCP | UDP |
| :--- | :--- | :--- |
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠 | 不可靠 |
| 速度 | 较慢 | 较快 |
| 开销 | 20-60 字节 | 8 字节 |

## 五、HTTP/HTTPS

### HTTP 方法

- GET：获取资源
- POST：提交数据
- PUT：更新资源
- DELETE：删除资源
- HEAD：获取头部
- OPTIONS：获取支持的方法

### 状态码

- 1xx：信息性
- 2xx：成功（200 OK）
- 3xx：重定向（301 永久重定向）
- 4xx：客户端错误（404 Not Found）
- 5xx：服务器错误（500 Internal Error）

### 请求头/响应头

- Content-Type、Content-Length
- Cookie、Set-Cookie
- Authorization

### HTTPS

- 加密方式：非对称加密 + 对称加密
- 端口：默认 443

## 六、Cookie / Session / Token

- **Cookie**：存储在客户端的文本信息
- **Session**：服务端存储的会话信息
- **Token**：无状态的令牌（如 JWT）

## 七、DNS 与跨域

- **DNS**：域名解析为 IP 地址
- **跨域**：浏览器同源策略限制，CORS 是解决方案

## 八、网络基础概念

### URL 特殊符号

- `#`：位置标识符，改变不重新加载
- `?`：连接作用，参数开始
- `&`：不同参数的分隔符

### 常用协议

- **ARP**：IP 地址转换为 MAC 地址
- **ICMP**：控制消息和错误报告（ping）
- **DHCP**：自动分配 IP 地址
