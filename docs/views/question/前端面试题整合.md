## 一、HTML

### 1.1 行内元素与块级元素

| 类型 | 特点 | 常见标签 |

## 二、CSS

### 2.1 盒模型

**W3C 标准盒模型**：
```
宽度 = content (内容宽度)
总宽度 = content + padding + border + margin
```

**IE 怪异盒模型**：
```
宽度 = content + padding + border
总宽度 = width + margin
```

**切换方式**：
```css
box-sizing: content-box;  /* W3C 标准 */
box-sizing: border-box;   /* IE 怪异模式 */
```

### 3.4 闭包

**概念**：有权访问另一个函数作用域中变量的函数。

**特点**：

### 3.5 原型与原型链

**原型**：
- 每个函数都有 `prototype` 属性（显式原型）
- 每个对象都有 `__proto__` 属性（隐式原型）

**原型链**：
- 查找属性时，若对象本身没有，会沿 `__proto__` 向上查找
- 直到 `Object.prototype.__proto__ === null`

```js
function Person(name) {
  this.name = name
}

Person.prototype.sayHello = function() {
  console.log(`Hello, ${this.name}`)
}

const p = new Person('Tom')
p.sayHello()  // Hello, Tom
```

### 3.6 深拷贝与浅拷贝

**浅拷贝**：只复制第一层属性
```js
Object.assign({}, obj)
{...obj}
```

**深拷贝**：递归复制所有层级
```js
// 方法一：JSON（无法拷贝函数）
JSON.parse(JSON.stringify(obj))

// 方法二：递归实现
function deepClone(obj) {
  if (obj === null || typeof obj !== 'object') return obj
  if (obj instanceof Array) return obj.map(item => deepClone(item))
  const clone = {}
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      clone[key] = deepClone(obj[key])
    }
  }
  return clone
}
```

### 3.9 事件循环（Event Loop）

**执行顺序**：
1. 执行同步代码（调用栈）
2. 执行微任务队列（Microtask）
3. 执行宏任务队列（Macrotask）

**微任务**：`Promise.then`, `MutationObserver`, `process.nextTick`
**宏任务**：`setTimeout`, `setInterval`, `I/O`, `UI渲染`

**示例**：
```js
console.log('1')  // 同步

setTimeout(() => {
  console.log('2')  // 宏任务
}, 0)

Promise.resolve().then(() => {
  console.log('3')  // 微任务
})

console.log('4')  // 同步

// 输出顺序：1 -> 4 -> 3 -> 2
```

### 3.12 节流与防抖

**防抖**：事件停止触发后执行一次
```js
function debounce(fn, delay) {
  let timer = null
  return function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => fn.apply(this, args), delay)
  }
}
// 应用：搜索输入框
```

**节流**：固定时间内只执行一次
```js
function throttle(fn, delay) {
  let lastTime = 0
  return function(...args) {
    const now = Date.now()
    if (now - lastTime >= delay) {
      lastTime = now
      fn.apply(this, args)
    }
  }
}
// 应用：滚动加载、按钮防重复点击
```

### 4.8 Vuex / Pinia

```

    { test: /\.vue$/, use: 'vue-loader' }
  ]
}
```

**Plugin**：扩展功能
```js
plugins: [
  new HtmlWebpackPlugin(),
  new CleanWebpackPlugin()
]
```

### 5.2 Vite vs Webpack

| 特性 | Vite | Webpack |
|------|------|---------|
| 启动速度 | 快（ES Module） | 慢（打包） |
| HMR | 即时更新 | 较慢 |
| 构建方式 | 按需加载 | 全量打包 |
| 适用场景 | 现代浏览器 | 兼容旧浏览器 |

## 六、WebSocket

### 6.1 简介

WebSocket 是 HTML5 提供的全双工通信协议，只需一次握手即可建立持久连接。

### 6.2 API

```js
const ws = new WebSocket('ws://localhost:8080')

// 连接建立
ws.onopen = () => {
  ws.send('Hello Server')
}

// 接收消息
ws.onmessage = (event) => {
  console.log('收到:', event.data)
}

// 连接关闭
ws.onclose = () => {
  console.log('连接关闭')
}

// 错误处理
ws.onerror = (error) => {
  console.error('错误:', error)
}
```

### 6.3 状态码

| 状态码 | 含义 |
|--------|------|
| `0` | 正在连接 |
| `1` | 连接成功 |
| `2` | 正在关闭 |
| `3` | 连接关闭 |

### 7.1 浏览器渲染流程

1. **解析 HTML** → 构建 DOM 树
2. **解析 CSS** → 构建 CSSOM 树

### 7.2 跨域解决方案

| 方案 | 原理 | 适用场景 |
|------|------|----------|
| **CORS** | 服务端设置响应头 | 后端可控 |
| **JSONP** | 动态创建 script | 仅 GET 请求 |
| **代理服务器** | 同源策略只限制浏览器 | 开发环境 |
| **WebSocket** | 不受同源限制 | 实时通信 |

**CORS 响应头**：
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type
```
